# Fast MCP

- [FastMCP Github](https://github.com/PrefectHQ/fastmcp)

## Servers

以下のコンポーネントを含むMCP(Model Context Protocol)サーバ。  

- Tools
- Resources
- Resource Templates
- Prompts

AI エージェントに直接搭載していた機能を外に出して再定義することで、AI エージェント間で機能を共有できるという代物。  
MCPに準拠したエージェントであれば、自分で用意したMCPサーバを接続することも出来る。  
例えば Claude に自分のソースコードを提供する MCP サーバなど。

### Core Components

#### Tools

いわゆる Tool Calling を公開する機能。  
LLM に直接公開して利用されることも出来るし、エージェント側で操作することも出来る。

#### Resources & Resource Templates

リソースとは URL ベースで公開するデータを指す。  
リソーステンプレートで公開した URL パターンに準拠して、リクエストが来たら該当するコンテンツを返すという仕組み。  
例) /users/{user_id}/profile

LLM にリソーステンプレートを公開することで、Tool の1つとして直接利用する事もあるし、エージェント側で操作して利用することも出来る。

#### Prompts

再利用可能なプロンプトを提供する。  
基本的にはパラメータを渡してプロンプトをビルドしてもう機能と考えて良い。  
LLM に直接利用させるというよりは、エージェント側で利用するものであると考える方が良いだろう。  
例) LLM の役割(ロール)を指定するためのテンプレート集。

#### Context

CodeIgniter で言うところの、`$CI` オブジェクトみたいなものをさす。  
ログの記録からセッション管理まで、コアな機能を搭載しているクラスであり、依存注入にて参照できる。

### Features

#### Background Tasks

文字通りの機能であり、時間のかかる処理をバックグラウンドで実行する機能である。  
進捗報告的な機能もあるらしい。

#### Composition

大雑把に言えば、子MCPを繋げるといった構成が可能である。  
子MCPに専門性の高い機能を持たせて、親MCPは窓口業務に専念するといった構成も可能なはずである。

#### Dependencies

いわゆる依存性の注入機能があり、タイプヒントをするだけで予約済のオブジェクトが自動的に注入される。  
いわゆる Laravel のサービスコンテナに近いものである。

- [Docket](https://github.com/chrisguidry/docket)

この注入機能は Docket が提供するものであるらしい。  
カスタム依存関係も作れるらしい。

#### Elicitation

いわゆる Human in the loop 機能も提供している。  
ツール実行中などに質問を返すことが出来るわけだが、あくまでエージェントの管理下によるツール利用に用途が限られるはずである。  
エージェントが特定のタスクをMCPに丸投げするようなケースで出番がありそう。

#### Icons

エージェントアプリに機能をアイコンとして表示するための機能。  
ツールのON/OFFなどのUIを提供する場合に利用できるだろう。

#### Lifespan

FastMCP サーバ起動・終了時にコールされるフックが提供されている。  
大き目な設定ファイルを起動時に全て読み込んでおくなどの利用方法がある。

#### Logging

MCPサーバのログではなく、クライアント側に送るログ機能がある。

#### Middleware

Laravel や Express といったミドルウェアと同じ概念ではあるが、基本的にグローバルな設定になる。  
特定のツールだけにミドルウェアを付けるといった実装は出来ない。

そのため特定のフック(メソッド)に処理を書くといった利用方法になる。  
例) on_call_tool フックはツールが実行された場合に呼び出される。

#### Pagination

大きなレスポンスを一度に返すのではなく、分割して返すページネーション機能がある。  
LLM が直接これを理解できるかどうかは微妙なところではあるが、出来そうな気もする。

#### Progress

ログと同じようなもので、クライアント側に状態を通知するための機能がある。

#### Sampling

MCP側からエージェントが管理しているLLMを利用するという機能。  
だいぶカオスな機能ではあるが、特定のMCP(ツール)に処理を丸投げしてしまうような場合に利用できそうではある。  
そのため、LLMからの tool calling も受け付けられるようである。

#### Storage Backends

キャッシュやセッションといったデータを永続保存するための機能。  
開発中はサーバのメモリで十分だが、本番では Redis を使うおいといったお話。

#### Telemetry

OpenTelemetry という規格で、サーバ内のトレース情報を送信できる。  
デバッグ目的にも使えるし、本番での不具合の追跡にも使える。  
ただしフルに機能を使う場合は、それなりに情報量が多いため、コスト(課金)もかかるだろう。

#### Test

Pytest を使ってテストを書こうというお話。

#### Versioning

各ツールにバージョンを付与して、バージョン別にツールを提供する機能。  
Python にはオーバーロードがないけど、同じ名前の関数を2重定義しても Syntax Error にはならないようである。  
とはいえ IDE は赤くなる可能性がある。
